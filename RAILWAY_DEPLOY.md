# Railway Deployment Guide

This project is fully compatible with Railway.app and leverages your `docker-compose.yml` to automatically provision all required services.

## Prerequisites
1. A Railway account (https://railway.app)
2. Railway CLI installed (`npm i -g @railway/cli`)
3. Gemini API Key

## Deployment Steps

1. **Login to Railway**
   ```bash
   railway login
   ```

2. **Initialize Project**
   ```bash
   railway init
   ```
   Select "Empty Project" or create a new one.

3. **Deploy the Stack**
   Railway natively supports `docker-compose.yml`. You can deploy the entire stack at once by running:
   ```bash
   railway up
   ```
   
   *Note: Railway will parse your `docker-compose.yml` and create separate services for PostgreSQL, Qdrant, Redis, MinIO, Backend, Worker, and Frontend.*

4. **Set Environment Variables**
   Once the services are created in your Railway dashboard, you need to add your API keys to the **backend**, **worker**, and **frontend** services:
   
   **For Backend & Worker:**
   - `GEMINI_API_KEY`: Your Google Gemini API Key
   - `GEMINI_LLM_MODEL`: `gemini-2.5-flash`
   - `GEMINI_EMBED_MODEL`: `text-embedding-004`
   - *Ensure the database connection URLs match the internal URLs provided by Railway.*

   **For Frontend:**
   - `NEXT_PUBLIC_API_URL`: The public URL generated by Railway for your backend service (e.g., `https://backend-production-xyz.up.railway.app/api/v1`).

5. **Expose Public Domains**
   Go to the Railway Dashboard:
   - Click on the **frontend** service -> Settings -> Networking -> Generate Domain.
   - Click on the **backend** service -> Settings -> Networking -> Generate Domain.

6. **Scale & Optimize**
   Since you are using Gemini via API, your backend no longer requires high RAM or GPU. You can confidently run the backend and frontend on standard Railway containers (512MB - 1GB RAM) without bottlenecking.
